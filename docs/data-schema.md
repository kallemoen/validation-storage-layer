# Validation & Storage Layer — Data Schema

This document defines every table, field, type, and constraint in the database. It is the contract that the Validation layer enforces — this is what "valid" means.

---

## Tables Overview

| Table | Purpose | Retention |
|---|---|---|
| `listings` | Stored property listings | Permanent |
| `scraper_registry` | Scraper configurations and health | Permanent |
| `currencies` | ISO 4217 currency lookup | Permanent |
| `run_receipts` | Per-run execution logs | 90 days |
| `rejections` | Failed validation records | 30 days |

---

## listings

The core table. One row per unique property listing.

### Metadata Fields

| Field | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `listing_id` | UUID | No | — | Primary key. Generated by the scraper. |
| `source_url` | VARCHAR(2000) | No | — | URL the listing was scraped from. **Unique index** — duplicates are rejected. |
| `config_id` | UUID | No | — | FK to `scraper_registry`. Which scraper found this listing. |
| `listing_type` | VARCHAR(20) | No | — | `sale` or `rent`. |
| `rent_period` | VARCHAR(20) | Yes | — | `monthly`, `weekly`, or `daily`. **Required when `listing_type = 'rent'`**, must be null when `sale`. |
| `listing_status` | VARCHAR(20) | No | `active` | `active`, `sold`, `delisted`, or `expired`. |
| `created_at` | TIMESTAMPTZ | No | `now()` | When first stored. Never modified. |
| `updated_at` | TIMESTAMPTZ | No | `now()` | Last modification time. |

### Location Fields

| Field | Type | Nullable | Description |
|---|---|---|---|
| `country_code` | CHAR(2) | No | ISO 3166-1 alpha-2 code (e.g., `PT`, `US`, `GB`). |
| `admin_level_1` | VARCHAR(255) | Yes | Top-level subdivision (state, district, region). |
| `admin_level_2` | VARCHAR(255) | Yes | Second-level subdivision (county, concelho, department). |
| `admin_level_3` | VARCHAR(255) | Yes | City / municipality / commune. |
| `admin_level_4` | VARCHAR(255) | Yes | Neighborhood / suburb / ward. |
| `postal_code` | VARCHAR(20) | Yes | Local format postal code. |
| `address_line_1` | VARCHAR(500) | Yes | Primary street address. |
| `address_line_2` | VARCHAR(500) | Yes | Secondary address line. |
| `latitude` | DECIMAL(9,6) | Yes | Actual property coordinates. Null if not provided by source. |
| `longitude` | DECIMAL(9,6) | Yes | Actual property coordinates. Null if not provided by source. |
| `display_latitude` | DECIMAL(9,6) | No | Always populated. For map rendering. |
| `display_longitude` | DECIMAL(9,6) | No | Always populated. For map rendering. |
| `location_granularity` | VARCHAR(20) | No | How precise the location data is (see enum below). |

**Admin level mapping by country:**

| Country | `admin_level_1` | `admin_level_2` | `admin_level_3` | `admin_level_4` |
|---|---|---|---|---|
| Portugal | Distrito | Concelho | Freguesia | Localidade |
| US | State | County | City | Neighborhood |
| UK | Country (England...) | County | City/Town | Ward |
| France | Region | Departement | Commune | Quartier |
| Japan | Prefecture | — | City | Ward |
| Brazil | State | — | Municipality | District |
| UAE | Emirate | — | City | Area/Zone |

### Pricing Fields

| Field | Type | Nullable | Description |
|---|---|---|---|
| `price_amount` | BIGINT | Yes | Current price in **minor units** (e.g., cents). Null = "price on request". |
| `price_currency_code` | CHAR(3) | Yes | ISO 4217 code. FK to `currencies`. Must be present when `price_amount` is present, and null when it's null. |
| `price_scraped_at` | TIMESTAMPTZ | Yes | When the current price was last observed. |
| `price_history` | JSONB | No | `[]` | Array of previous price observations. |

**Price history format:**

```json
[
  { "amount": 180000, "currency_code": "EUR", "scraped_at": "2026-03-01T10:00:00Z" },
  { "amount": 175000, "currency_code": "EUR", "scraped_at": "2026-02-15T08:00:00Z" }
]
```

**Minor units:** Prices are stored as integers to avoid floating-point precision issues. Divide by `10^minor_units` (from the `currencies` table) to get the display value. For EUR (2 minor units): `170000` = 1,700.00 EUR.

### Rooms & Size Fields

| Field | Type | Nullable | Description |
|---|---|---|---|
| `bedrooms` | SMALLINT | Yes | Number of bedrooms. |
| `bathrooms` | DECIMAL(3,1) | Yes | Number of bathrooms. Supports half-baths (e.g., `2.5`). |
| `total_rooms` | SMALLINT | Yes | Total rooms as defined by local market convention. |
| `living_area_sqm` | DECIMAL(10,2) | Yes | Living area in square meters. |
| `plot_area_sqm` | DECIMAL(12,2) | Yes | Land/plot area in square meters. |
| `raw_room_description` | VARCHAR(500) | Yes | Exact source text (e.g., `"3 pièces"`, `"3LDK"`, `"T2 - 2 quartos"`). |

### Property Type Fields

| Field | Type | Nullable | Description |
|---|---|---|---|
| `property_type` | VARCHAR(20) | No | Fixed enum for cross-market filtering. |
| `property_subtype` | VARCHAR(100) | Yes | Market-specific subtype (e.g., `single_family`, `condo`, `riad`). |
| `raw_property_type` | VARCHAR(500) | Yes | Exact source text. |

### Content Fields

| Field | Type | Nullable | Description |
|---|---|---|---|
| `title` | VARCHAR(500) | No | Listing title or headline. |
| `description` | TEXT | Yes | Raw description text. Any language, any length. |
| `images` | JSONB | Yes | Ordered array of image URLs. First entry is the hero image. |
| `raw_data` | JSONB | No | Complete raw extraction output. Write-once. Structure varies per source. |

### Constraints

- **Unique source_url:** `CREATE UNIQUE INDEX idx_listings_source_url ON listings(source_url)`
- **Rent period required for rentals:** `(listing_type = 'rent' AND rent_period IS NOT NULL) OR (listing_type = 'sale' AND rent_period IS NULL)`
- **Price/currency pair:** `(price_amount IS NOT NULL AND price_currency_code IS NOT NULL) OR (price_amount IS NULL AND price_currency_code IS NULL)`

### Indexes

| Index | Columns |
|---|---|
| `idx_listings_source_url` | `source_url` (unique) |
| `idx_listings_config_id` | `config_id` |
| `idx_listings_country` | `country_code` |
| `idx_listings_status` | `listing_status` |
| `idx_listings_type` | `listing_type` |
| `idx_listings_created` | `created_at` |

## scraper_registry

One row per scraper configuration. Tracks scraper health and operational state.

| Field | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `config_id` | UUID | No | `gen_random_uuid()` | Primary key. Auto-generated on registration. |
| `agency_name` | VARCHAR(255) | No | — | Human-readable agency name (e.g., `remax.pt`). |
| `country_code` | CHAR(2) | No | — | Market this scraper covers. |
| `area_key` | VARCHAR(255) | No | — | Area within the market (e.g., `Lisboa`). |
| `listing_type` | VARCHAR(20) | No | — | `sale` or `rent`. |
| `status` | VARCHAR(20) | No | `testing` | `active`, `paused`, `broken`, `degraded`, or `testing`. |
| `created_at` | TIMESTAMPTZ | No | `now()` | When first registered. |
| `last_run_at` | TIMESTAMPTZ | Yes | — | Last time the scraper was run. |
| `last_run_status` | VARCHAR(20) | Yes | — | `success`, `partial`, or `failure`. |
| `last_run_listings` | INTEGER | Yes | — | Listings accepted on the last run. |
| `failure_count` | SMALLINT | No | `0` | Consecutive failed runs. Resets on success. |
| `broken_at` | TIMESTAMPTZ | Yes | — | When the scraper was marked as broken. |
| `repair_count` | SMALLINT | No | `0` | How many times the scraper has been repaired. |
| `expected_discovery_count` | INTEGER | No | `0` | Exact number of listings discovery should return. Used for health check 1b (discovery mismatch). |
| `run_interval_hours` | INTEGER | No | `24` | How often the scraper runs, in hours. Used for health check 6 (scraper gone silent). |
| `last_successful_insert_at` | TIMESTAMPTZ | Yes | — | When a listing was last successfully inserted. Used for health check 5 (no new listings). |
| `status_reason` | VARCHAR(255) | Yes | — | Human-readable reason for the current status (e.g., `"Discovery mismatch: 85 found, expected 100"`). Set to `"Manually set to {status}"` on manual status change. |
| `config` | JSONB | No | — | Full scraper configuration. Structure varies per scraper. Must include `currency_code`. |

**Automatic behaviors:**
- After 3 consecutive failures (`failure_count >= 3`), status is automatically set to `broken`.
- When moving from `broken`/`testing` to `active`, `repair_count` is incremented and `failure_count` is reset.

## currencies

ISO 4217 currency lookup table. Pre-seeded with common currencies.

| Field | Type | Nullable | Description |
|---|---|---|---|
| `code` | CHAR(3) | No | Primary key. ISO 4217 code (e.g., `EUR`, `USD`). |
| `name` | VARCHAR(100) | No | Human-readable name. |
| `symbol` | VARCHAR(5) | No | Display symbol (e.g., `€`, `$`). |
| `minor_units` | SMALLINT | No | Number of decimal places. Divide `price_amount` by `10^minor_units` for display. |

**Seeded currencies:** EUR, USD, GBP, BRL, JPY, AED, MAD, CHF, SEK, NOK, DKK, PLN, CZK, HUF, TRY, ZAR, AUD, CAD, MXN, INR.

## run_receipts

One row per scraper pipeline run. Provides operational visibility into scraper performance.

| Field | Type | Nullable | Description |
|---|---|---|---|
| `id` | UUID | No | Primary key. Auto-generated. |
| `config_id` | UUID | No | FK to `scraper_registry`. |
| `started_at` | TIMESTAMPTZ | No | When the run started. |
| `completed_at` | TIMESTAMPTZ | No | When the run finished. |
| `status` | VARCHAR(20) | No | `success`, `partial`, or `failure`. |
| `failure_stage` | VARCHAR(20) | Yes | Which stage failed: `discovery`, `extraction`, or `validation`. |
| `urls_discovered` | INTEGER | Yes | Total URLs found during discovery. |
| `urls_new` | INTEGER | Yes | URLs not already in the database. |
| `listings_extracted` | INTEGER | Yes | Listings successfully extracted from pages. |
| `listings_submitted` | INTEGER | Yes | Listings submitted for validation. |
| `listings_accepted` | INTEGER | Yes | Listings that passed validation. |
| `listings_rejected` | INTEGER | Yes | Listings that failed validation. |
| `error_message` | TEXT | Yes | Error details if the run failed. |

**Retention:** Automatically purged after 90 days via `pg_cron`.

---

## rejections

One row per rejected listing. Stores the full submitted data and detailed validation feedback for troubleshooting.

| Field | Type | Nullable | Description |
|---|---|---|---|
| `id` | UUID | No | Primary key. Auto-generated. |
| `config_id` | UUID | No | FK to `scraper_registry`. |
| `mode` | VARCHAR(10) | No | `test` or `live`. |
| `listing_data` | JSONB | No | The complete listing object that was submitted. |
| `tier_1_errors` | JSONB | No | Array of schema validation errors. |
| `tier_2_errors` | JSONB | No | Array of semantic validation errors. |
| `tier_3_warnings` | JSONB | No | Array of completeness warnings. |
| `created_at` | TIMESTAMPTZ | No | When the rejection occurred. |

**Retention:** Automatically purged after 30 days via `pg_cron`.

## Enum Reference

### listing_type
`sale`, `rent`

### rent_period
`monthly`, `weekly`, `daily`

### listing_status
`active`, `sold`, `delisted`, `expired`

### property_type
`house`, `apartment`, `land`, `commercial`, `mixed_use`, `parking`, `other`

### location_granularity

Describes how precise the location data is. Ordered from most to least granular:

| Value | Meaning | display coordinates are... |
|---|---|---|
| `coordinates` | Exact lat/lng provided | Exact property location |
| `address` | Street address provided, geocoded | Geocoded address |
| `postal_code` | Postal code provided | Random point within postal code area |
| `admin_level_4` | Neighborhood known | Random point within neighborhood |
| `admin_level_3` | City known | Random point within city |
| `admin_level_2` | County/district known | Random point within district |
| `admin_level_1` | State/region known | Random point within region |
| `country` | Only country known | Random point within country |

### scraper status
`active`, `paused`, `broken`, `degraded`, `testing`

### run status
`success`, `partial`, `failure`

### failure_stage
`discovery`, `extraction`, `validation`
